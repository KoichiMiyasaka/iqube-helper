## 1. このツールは何？

iQube の勤怠入力を「ワンクリックで終わらせる」ためのブックマークレット。

- PC（Chrome / Edge / Firefox / Safari）と iPhone Safari に対応
- 個別ユーザーごとに iQube アカウントは必要（管理者権限は不要）
- サーバー不要、フロントエンドのみで完結

### 主な機能

| 機能 | 内容 |
|------|------|
| 今日を打刻 | ボタンを押した日の勤怠を即入力 |
| 月一括打刻 | 選択した月の平日全部を一括入力 |
| 日付選択一括 | カンマ区切りまたは範囲指定で複数日入力 |
| 定時カスタマイズ | 端末ごとに出社・退社時刻を変更可能 |
| 備考スキップ | 有給休暇など備考が書かれた日は自動回避 |
| 祝日スキップ | 土日・祝日は自動で除外 |
| 未来日ブロック | 明日以降は打刻できない |

### デフォルト定時

| 項目 | 時刻 |
|------|------|
| 出社 | 09:00 |
| 退社 | 18:00 |
| 外出（休憩開始） | 12:00 |
| 戻り（休憩終了） | 13:00 |

→ ブラウザの `localStorage` に保存して個別カスタマイズ可能

## 2. なぜ作ったか

### 課題

- iQube の勤怠入力は1日ずつ手動入力が必要
- 1ヶ月分まとめてやろうとすると 約20日 × 4項目（出社・退社・外出・戻り） = **80回のフォーム入力 + 80回のクリック** が発生
- 月末・月初のタイミングで毎回 **15〜20分のロス**
- 「入力忘れ」が定期的に発生し、リカバリも面倒

### 解決アプローチ

- iQube の既存 API を流用するブックマークレット形式に
  → ブラウザ拡張やデスクトップアプリ不要、配布が楽
- GitHub Pages から `loader.js` を動的ロード
  → ブックマークを更新せず、コード修正が全員に即反映される
- 既存のログインセッションを再利用
  → ID/PW を一切扱わない（盗難リスクゼロ）

## 3. 技術スタック

### 使っている技術

| 技術 | 用途 |
|------|------|
| Vanilla JS (ES6) | 全実装（依存ライブラリゼロ） |
| Bookmarklet | 配布形式 |
| GitHub Pages | `loader.js` のホスティング |
| Fetch API | iQube への HTTP リクエスト |
| DOMParser | iQube レスポンスHTMLからのトークン抽出 |
| localStorage | ユーザー個別の定時設定保存 |

- 意図的に「フレームワーク・ビルドツールゼロ」で構成
- 1ファイル（`loader.js`）で完結 → 学習コスト最小

### ファイル構成

```
iqube-helper/
├── index.html      # 配布ページ（GitHub Pages 公開）
├── loader.js       # 本体（約700行、全機能を含む）
└── README.md       # ドキュメント
```

## 4. アーキテクチャ

### 全体フロー

```
[ユーザー] ─クリック→ [ブックマークレット]
                       │
                       ↓
                   loader.js を <script> として動的注入
                       │
                       ↓
            [GitHub Pages] ──loader.js を配信→ [ブラウザ]
                       │
                       ↓
                   UI パネルが iQube ページに重ねて表示
                       │
                       ↓
               ユーザーが日付・時刻を指定して実行
                       │
                       ↓
              [iQube API] ←PUT /time_cards (CSRF token 付き)
```

### ブックマークレットURL

```javascript
javascript:(()=>{
  const s=document.createElement('script');
  s.src='https://koichimiyasaka.github.io/iqube-helper/loader.js?v='+Date.now();
  document.body.appendChild(s);
})();
```

**ポイント**:
- `?v=Date.now()` でキャッシュバスティング → 最新版を毎回取得
- 1行で完結（ブックマーク制限内）

## 5. 設計上の工夫

### ① CSRF トークン取得の動的化

iQube は `<meta csrf-token>` をページに埋め込まない仕様。

→ 「編集モーダルを開いた時にサーバーから返るHTML」内の `<input name="authenticity_token">` から動的に抽出。

さらに、属性順がランダム化されている可能性に備えて **正規表現を7パターン** 用意し、DOMParser フォールバックも実装。

```javascript
// loader.js より抜粋
const patterns = [
  /name="authenticity_token"[^>]*value="([^"]+)"/i,
  /value="([^"]+)"[^>]*name="authenticity_token"/i,
  /<meta name="csrf-token"[^>]*content="([^"]+)"/i,
  /"authenticity_token"\s*:\s*"([^"]+)"/i,
  // ...
];
```

### ② 祝日判定の動的化

当初は祝日リストをハードコードしていたが、年次メンテが面倒。

→ iQube画面で赤色表示されている `td.holiday` クラスから動的取得に変更。

**メリット**:
- 2028年問題（祝日法改正）対応不要
- 会社独自の休日（年末年始・創立記念日）も自動対応
- iQube の判定基準と完全一致

### ③ 備考あり日の自動スキップ

有給休暇・出張・欠勤など、備考が入っている日は誤って上書きされると困る。

→ `bulkPunch` 時に各日の current state を取得し、備考フィールドが空でない日は skip log に記録してスキップ。

### ④ GitHub Pages からの動的ロード

ブックマークレットを再登録せず、全員に最新版を配布できる仕組み。

- `?v=Date.now()` でキャッシュバスティング
- `loader.js` を git push するだけで、数分後に全員へ反映

### ⑤ 個別ユーザーの定時カスタマイズ

社内全員のデフォルトは `DEFAULT_TIMES`（`loader.js` 冒頭）で管理。
個別ユーザーが時刻を変えたい場合は localStorage に保存。

→ 「全社統一の更新」と「個人カスタマイズ」を両立

## 6. セキュリティ設計

### 認証情報の扱い

- このツールは **ID/PW を一切扱わない**
- ブラウザに既存ログインセッションがあることを前提
- iQube のセッションクッキーをそのまま利用

→ 万が一コードに脆弱性があっても、認証情報の漏洩リスクはゼロ

### XSS・CSRF 対策

- CSRF token は iQube の正規エンドポイントから取得
- `loader.js` は GitHub Pages（HTTPS）からのみ配信
- 第三者のコード注入経路は存在しない

### 操作ログ

- 全打刻は iQube 側のサーバーログに記録される
- 「誰が」「いつ」「どの日を」打刻したかは iQube 監査ログで追跡可能
- ツール側でログを別途持つことはしない（個人情報を増やさない方針）

## 7. 開発・運用フロー

### 開発

```bash
git clone git@github.com:KoichiMiyasaka/iqube-helper.git
cd iqube-helper
# loader.js を編集
git add . && git commit -m "..." && git push origin main
```

- push 後、GitHub Pages が数分で反映
- 全員のブックマーク（`?v=Date.now()` 付き）が次回起動時に最新版取得

### デフォルト定時の変更（管理者向け）

`loader.js` の冒頭 `DEFAULT_TIMES` を編集して push。
全員に反映されるが、localStorage で個別変更している人には適用されない。

### コミット履歴（参考）

1. `feat:` 初期実装 - iQube クイック打刻ブックマークレット
2. `fix:` CSRFトークンを編集モーダルHTMLから動的取得する
3. `fix:` トークン抽出ロジックを強化（7パターンの正規表現+DOMParser fallback）
4. `feat:` 祝日スキップに対応（2025-2027年の日本の祝日を内蔵）
5. `docs:` 祝日リストの更新手順をREADMEに追記
6. `feat:` 備考欄が入っている日を一括打刻時にスキップする
7. `docs:` 備考スキップ機能の説明を追加
8. `refactor:` 祝日判定をiQube画面のtd.holidayクラスから動的取得に変更

## 8. なぜブックマークレット形式にしたか

### 代替案の比較

| 方式 | 配布容易 | 更新容易 | 認証安全 | 動作信頼 |
|------|:---:|:---:|:---:|:---:|
| Chrome拡張機能 | △ | ○ | ○ | ◎ |
| デスクトップアプリ | × | × | △ | ○ |
| Tampermonkey | △ | ○ | ○ | ○ |
| **ブックマークレット** | **◎** | **◎** | **◎** | **○** |
| サーバー型ツール | ◎ | ◎ | × | ○ |

### ブックマークレット採用の決め手

- インストール不要（ドラッグ&ドロップのみ）
- Chrome拡張のような審査・公開手続き不要
- GitHub Pages からの動的ロードで更新即時反映
- サーバー不要 → 認証情報・打刻データを第三者サーバーに送らない

## 9. 制約・既知の課題

### 制約

- iQube のドメイン（`*.iqube.net`）でのみ動作
- iPhone Safari 以外のモバイルブラウザは未検証
- iQube の UI 仕様変更時にメンテナンスが必要

### 既知の課題

- 完全自動化（cron的に毎日19時に打刻）は別途設定が必要
  → README に macOS/Windows 自動化手順を追記済み
- 残業時間や深夜手当の入力は非対応（手動）
- 出張・テレワーク区分などの選択肢は非対応（手動）
